## מטרה
לשדרג את `מודל ברטל - שאלון שחקן.tsx` לרמה מקצועית: פיקסל-ארט אמיתי באיכות גבוהה,

## החלטות שהתקבלו (Q&A)
- **סגנון אומנות:** פיקסל-ארט אמיתי דרך Aseprite, בתהליך **היברידי** — רפרנס AI (Gemini) לקביעת הרכב/פרטים → בנייה מחדש כפיקסל-ארט אמיתי ב-Aseprite (קוונטום פלטה, הצמדה לרשת, ניקוי ידני). התוצאה הסופית היא פיקסל-ארט אמיתי (פלטה מוגבלת, רשת חדה).
- **הטמעה:** ייצור מראש + הטמעה כ-base64 ב-tsx → עובד אופליין, ללא תלות ב-API.
- **היקף אומנות:** 4 דיוקני תוצאה + 4 אייקונים למסך פתיחה.
- **שאלות/סיכום:** כתיבה מחדש מלאה.
- **תוצרים:** גם `.tsx` וגם `index.html` עצמאי לתצוגה בדפדפן.
- **קהל יעד:** חינוכי / נוער — שפה נגישה, פחות ז'ארגון גיימינג.

## ארכיטקטורה ורכיבים

### 1. צינור האומנות — היברידי (nano-banana לרפרנס → aseprite-art + pixel-art-professional לבנייה)
שלבים לכל דיוקן:
1. ייצור רפרנס מפורט עם Gemini (nano-banana) — דמות RPG בסגנון פיקסל, להרכב/פרטים/צבעוניות.
2. ייבוא ל-Aseprite, downscale לרשת 64×64, קוונטום פלטה (~16–24 צבעים), הצמדה לרשת.
3. ניקוי ידני: חידוד קצוות, ramps לצללית, hue-shifting, AA סלקטיבי — עד מראה מקצועי.
- **4 דיוקני תוצאה**, 64×64 native, דמויות חזה (ראש+כתפיים), סגנון 16-bit RPG:
  - **A — הכובש / מלך:** כתר זהב, שריון מלכותי, זהב חם + סגול.
  - **E — מגלה הארצות / סייר:** ברדס ירוק, מחזיק מפה/מצפן זוהר.
  - **S — חבר הלהקה / נגן:** נגן עליז עם נבל/לאוטה, גוונים חמים וידידותיים.
  - **K — הלוחם / מתנקש:** ברדס כהה, עיניים אדומות זוהרות, פגיונות.
- **4 אייקונים** 32×32 — גרסאות אמבלמה/פנים קטנות לשורת הפתיחה.
- ייצוא PNG ל-`assets/` + הטמעה כ-base64 ב-tsx וב-html.
- תצוגה חדה דרך CSS `image-rendering: pixelated`, מוגדל במסגרות מעוטרות.
- **נקודת בקרה:** רינדור → הצגת ה-PNG-ים למשתמש → ליטוש עד "מקצועי" → רק אז חיווט לקומפוננטה.

### 2. שינויי קומפוננטה
- החלפת `PixelGraphic` (SVG מטריצות) וקריאת `generateCharacterImage` ל-Gemini בזמן ריצה
  ברינדור `<img>` של פיקסל-ארט מוטמע.
- שמירת אנימציית "זימון" קוסמטית קצרה (~1.5ש') לפני חשיפת התוצאה (ללא המתנת API אמיתית).
- שמירת מבנה ה-state הקיים (`gameState`, `scores`, ערבוב אפשרויות).

### 3. שאלות — כתיבה מחדש מלאה
- 12 שאלות תרחיש (במקום 10) לרזולוציית טיפוס טובה יותר.
- כל שאלה: 4 אפשרויות מאוזנות הממופות ל-A/E/S/K, לא שקופות מדי.
- שפה נגישה לנוער, עברית RTL, הקשר חינוכי של גילוי עצמי. פחות ז'ארגון; מונחים לועזיים מוסברים.

### 4. סיכום שחקן — עיצוב עשיר
- דיוקן במסגרת מעוטרת + כותרת + תת-כותרת תיאורית.
- **טיפוס ראשי + משני** (למשל "כובש עם נטיית חוקר") מתוך שני הציונים הגבוהים.
- **פירוק ויזואלי באחוזים:** 4 פסים אופקיים בסגנון RPG (לא רק מספרים גולמיים).
- **נקודות חוזק** + **טיפ/המלצה** מותאמים לטיפוס.
- כפתור התחלה מחדש.

### 5. תוצרים
- `מודל ברטל - שאלון שחקן.tsx` משודרג (Tailwind, `dir="rtl"`).
- `index.html` עצמאי (React + Tailwind דרך CDN) — פתיחה בדפדפן וצפייה מיידית.
- `assets/` עם קבצי ה-PNG.

## זרימת נתונים
intro → playing (12 שאלות, צבירת scores A/E/S/K) → loading (אנימציה קצרה) →
result (חישוב top-1/top-2, אחוזים, דיוקן, חוזקות, טיפ) → restart.

## טיפול בשגיאות

- אם דיוקן חסר — fallback לאייקון הטיפוס.
